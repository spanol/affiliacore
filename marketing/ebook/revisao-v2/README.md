# Revisão v2 — pacote separado para análise antes de merge

Este diretório foi criado para concentrar **duas ondas de refinamento** do ebook da AffiliaCore **sem tocar no rascunho atual** em `../rascunho/`.

Objetivo:
- preservar o rascunho v1 intacto;
- analisar cortes, reestruturações e upgrades de conversão em separado;
- decidir depois o que entra no merge final.

## Estrutura

- `01-onda-enxugamento-estrutura.md`
  - foco: densidade, ritmo, escaneabilidade, capítulos mais pesados, cortes e reorganizações.
- `02-onda-conversao-progressao.md`
  - foco: promessa comercial, progressão ebook → próximo passo, concretude operacional, suavização do capítulo de produto e encerramento.

## Regras deste pacote

1. **Nenhum arquivo do rascunho atual foi editado aqui.**
2. As recomendações abaixo são de **análise/editoria**, não de merge automático.
3. O critério de decisão sugerido é:
   - aprovar / ajustar / descartar cada bloco;
   - só depois aplicar no conteúdo final.

## Leitura recomendada

1. Ler `01-onda-enxugamento-estrutura.md`
2. Ler `02-onda-conversao-progressao.md`
3. Marcar por capítulo:
   - o que entra sem discussão;
   - o que entra com adaptação;
   - o que não entra.

## Resultado esperado após merge

Se as duas ondas forem absorvidas com disciplina, o ebook deve sair de:
- **whitepaper institucional denso**

para algo mais próximo de:
- **tripwire sério, legível e progressivo**, com autoridade suficiente para sustentar o funil da AffiliaCore sem parecer só uma brochura de produto.
